# Patterns And Examples

Use these as starting patterns. Replace the placeholders with real facts.

## 1. Cinematic Concept Sketch

### Positive Prompt

```text
彩色电影概念精细草图，影视前期美术开发稿，不是海报，不是完成插画。画面保留清晰起稿线、结构辅助线、薄涂上色、试色痕迹和局部未收死的边缘。主体为[角色或对象描述]，构图为[半身/全身/远景]，[视角说明]，[不看镜头/看向远处/侧身观察]。第一眼先看到[主视觉焦点]，第二眼注意到[次级信息]，第三眼才意识到[隐藏的情绪或机制]。服饰、发型、妆造、配件要体现[文明/门第/职业/设计体系]，材料呈现[布料、金属、皮革、树脂等真实质感]。环境是[氛围背景/结构场景]，承担[情绪/叙事/尺度]职责。整体高级、克制、有开发稿感。
```

### Negative Prompt

```text
不要成品海报，不要手游卡面，不要背景画满，不要无意义大光效，不要古偶滤镜，不要甜美网红脸，不要把开发稿做成完成插画。
```

## 2. Clean Standing Illustration

### Positive Prompt

```text
干净克制的角色基准立绘，完整展示[角色名称或描述]的稳定设计语言，用于后续一致性复用。人物全身或接近全身，[正视/3/4 侧身]，[看镜头/不看镜头]，站姿[稳定/克制/警惕/松弛]。重点锁定脸型、发型、妆造、服装结构、腰线、层次关系、鞋靴、武器或法器位置。服饰必须清楚体现[设计系统]，不是模糊风格词堆砌。材质清楚，轮廓干净，背景极简或弱氛围，避免抢主体。整体不是海报，而是可供下游复用的基准角色图。
```

### Negative Prompt

```text
不要海报构图，不要随机大片背景，不要过量特效，不要现代时装乱入，不要和角色身份不匹配的妆造，不要把基准图画成情绪海报。
```

## 3. Chibi Figurine Product Shot

### Positive Prompt

```text
角色视觉商品展示图，同一画面同时出现二维角色立绘与基于该立绘制作的 Q版收藏级 3D 手办，高端商业产品摄影风格。画面主体是一尊高精度 PVC 或 ABS Q版手办，头身比压缩但不是廉价盲盒，不是幼态卖萌。手办站立在干净克制的半透明底座上，底座可带极淡纹理但不要文字。旁边摆放角色包装盒，盒面完整印刷该角色二维全身立绘。二维原画、Q版手办、包装盒必须是同一设计语言，同一服装结构，同一道具系统，同一发型与气质。背景为浅灰或冷白棚拍环境，柔和主光加轻微轮廓光，阴影真实，材质高级，可收藏。
```

### Negative Prompt

```text
不要廉价盲盒，不要过幼态，不要无脑卖萌，不要大眼萌系默认脸，不要包装盒与手办不是同一设计，不要杂乱背景，不要廉价景品感。
```

## 4. Realistic Figurine Development Shot

### Positive Prompt

```text
角色立绘加写实收藏手办的开发展示图，真实桌面环境中的高端产品摄影。一尊 1/7 比例收藏级 PVC 手办摆放在桌面展示底座上，旁边放角色包装盒，包装盒正面完整印刷该角色二维全身立绘。桌面显示器屏幕显示这尊手办的数字雕刻或 ZBrush 建模界面，形成二维原画到三维建模再到实体手办的完整信息链。手办本体必须高度还原包装盒上的二维原画，脸、发型、服装比例、道具位置完全统一。桌面环境真实但克制，带少量专业工作痕迹。整体为写实商业摄影，PVC 表面高光自然，底座反光真实，显示器冷光与棚灯补光配合，景深专业。
```

### Negative Prompt

```text
不要把显示器界面做成无关装饰，不要乱桌面，不要包装盒与手办风格脱节，不要廉价工作室摆拍，不要人物变成平面海报贴图，不要 Q版化。
```

## 5. Prop Or Equipment Concept Sheet

### Positive Prompt

```text
彩色概念设计图，前期设计页，不是海报。主体为[武器/法器/装备/配件]的概念设计，清楚展示整体轮廓、关键结构、材料分层、局部细节和使用痕迹。画面可包含主视图、局部放大、结构拆解、材质试色和简短的设计标注。重点是让观者看懂它的设计系统、使用逻辑和世界归属，而不是做成摆拍插画。线稿与上色并存，像高级影视或游戏前期开发稿。
```

### Negative Prompt

```text
不要海报，不要只会摆帅，不要看不清结构，不要纯特效堆叠，不要失去材料层次，不要把设计页画成成品宣传图。
```

## 6. Technical Infographic Or Architecture Explainer

### Positive Prompt

```text
干净现代的技术信息图 / 架构解释图，用于把[主题]清楚讲给初学者，不是海报，不是 UI 截图。画面先锁版式再谈风格：整体为[上下两区 / 四列结构 / 底部组件区]，第一眼先看到[标题]，第二眼看到[核心关系，例如 Host -> Client -> Server -> Resource]，第三眼才看到[底部组件或补充说明]。明确写出每个区域的职责、列关系和卡片数量，例如左列是[主机或来源]，中列是[桥接或客户端]，中右列是[服务或处理层]，右列是[目标资源或外部系统]。标签只保留短词或短语，尽量重复使用相同标签，不要塞段落。图标语言为扁平、圆角、轻描边、友好教育风格，背景干净留白充足，箭头关系清楚，配色为[浅绿 / 浅黄 / 浅蓝 / 浅紫等柔和配色]。整体像高质量教学信息图，而不是营销海报。
```

### Negative Prompt

```text
不要海报感，不要假 PPT 模板感，不要深色科技风，不要重渐变，不要大段文字，不要把多个资源区合并成一个大面板，不要乱加框，不要乱改箭头关系，不要难以辨认的小字。
```

### Note

If exact labels, spelling accuracy, or later editability are critical, use this
prompt to lock style and layout direction first, then rebuild the final diagram
in a vector or diagram tool instead of treating the raster result as final by
default.
